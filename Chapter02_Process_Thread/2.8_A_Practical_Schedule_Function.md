## ✅ 知识点1: Linux 0.11 的 `schedule()` 函数

**看看`schedule`具体如何实现的...**
- `schedule()` 函数定义于 `kernel/sched.c`，是 Linux 0.11 的进程调度核心
    ```c
    void Schedule(void)  // 在 kernel/sched.c 中
    {
        while(1)
        {
            c = -1;
            next = 0;
            i = NR_TASKS;
            p = &task[NR_TASKS];
            while(--i){
                if((*p->state == TASK_RUNNING && (*p)->counter > c)
                    c = (*p)->counter, next = i; //counter指优先级priority，本身也作为时间片进行轮转调度
            } 
            if(c) break;  // 找到了最大的 counter
            for(p = &LAST_TASK; p > &FIRST_TASK; --p)
                (*p)->counter = ((*p)->counter >> 1) + (*p)->priority;
        }
        switch_to(next);
    }
    ```
    - **核心逻辑**：遍历所有任务，找出 `counter` 值最大且状态为 `TASK_RUNNING` 的进程，切换执行
    - **如果 `c == 0`了则**：对所有进程执行 `((*p)->counter >> 1) + (*p)->priority`
        - **对于就绪态进程，说明其`counter`一定都为0了**：则重新将重置为初始值 `priority`
        - **对于阻塞态进程来说，其 `counter >0`**：通过重置的 `priority` 一定 `>` 就绪状态进程的初始值 
            > ⚠️ 也就是说那些指向IO的进程优先级会更高
            > 💡 **理解技巧**：阻塞态进程一定是在 `counter` > 0 的时候被切换去执行 IO 的
            > ⚠️ `counter` 既是"还能跑多久"的时间片，也是"值越大越优先"的调度权重
    - 最终调用 `switch_to(next)` 完成上下文切换
        - `NR_TASKS` = 最大任务数
        - `task` 是进程表，一个全局指针数组，每个元素指向一个 `task_struct`（TCB）
            > ⚠️ 现代 Linux 内核里已经没有 task[] 数组了，用的是红黑树/链表管理 `task_struct`

> ⚠️ **关键区分**：外层的 `while(1)` 确保一定选出一个进程——第一次循环选 `counter` 最大者，若全为0则重新计算后再选
> 💡 **理解技巧**：`if(c) break` 是核心短路逻辑，`c > 0` 说明找到了有剩余时间片的可运行进程

## ✅ 知识点2: `counter` 作为时间片

**`counter` 的时间片作用如何体现？**
- `counter` 是典型的时间片（`Time Slice`），调度本质是**轮转调度（`Round-Robin, RR`）**

- **`do_timer` 中修改 `counter`**
    ```c
    void do_timer(...)  // 在 kernel/sched.c 中
    {
        if((--current->counter > 0)
            return;
        current->counter = 0;
        schedule();
    }
    ```
    - 每次时钟中断调用 `do_timer()`，将当前进程 `counter` 减1
    - `counter` 减到0时触发 `schedule()`，切换下一个进程
    - 这**保证了响应时间的界**——每个进程最多等待一个时间片周期
- **时钟中断**：
    ```asm
    _timer_interrupt: //在 kernel/system_call.s
        ...
        call _do_timer
    ```

- **设置好时钟中断**：
    ```c
    void sched_init(void)
    {
        set_intr_gate(0x20, &timer_interrupt);  // 0x20 = 时钟中断
    }
    ```
    - **将时钟中断 `0x20` 的处理函数 `timer_interrupt` 以中断门形式写入 `IDT`**
    - **同时 `set_intr_gate` 进入时自动关中断防止防止时钟中断处理过程中再被另一个时钟中断打断。**

> ⚠️ **关键区分**：`do_timer()` 里 `--current->counter > 0` 是前缀递减，先减再判断
> 💡 **理解技巧**：时钟中断 → `timer_interrupt` → `do_timer()` → `counter--` → `counter == 0` → `schedule()`，这条链就是轮转调度的完整机制

## ✅ 知识点3: `counter` 作为优先级

**`counter` 的优先级作用如何体现？**
- `schedule()` 选 **counter 最大**的任务执行，因此 `counter` 同时表示**优先级（`Priority`）**
- 当所有进程 `counter` 都耗尽（`c == 0`）时，执行重新计算：
  $$c(t) = \frac{c_{t-1}}{2} + p$$
  其中 $p$ = `priority`（静态优先级），$c_{t-1}$ = 上一次剩余时间片
- **阻塞后再就绪的进程优先级高于非阻塞进程**：
  - 进程因 I/O 阻塞 → 不消耗 CPU → `counter` 不被递减 → 阻塞结束后 `counter` 相对较大
- **学习机制**：
  - I/O 时间越长 → `counter` 相对越大 → 被优先调度
  - **这正是前台进程的特征**（前台进程频繁 I/O），因此优先级就一定会比后台进程更高


    ```c
    // 调度选择：counter 最大的任务
    while(--i){
        if((*p->state == TASK_RUNNING && (*p)->counter > c)
            c = (*p)->counter, next = i;
    }

    // counter 重新计算：动态优先级调整
    for(p = &LAST_TASK; p > &FIRST_TASK; --p)
        (*p)->counter = ((*p)->counter >> 1) + (*p)->priority;
    ```


> ⚠️ **关键区分**：`counter >> 1` 是右移一位，等价于除以2，但比除法更快
> 💡 **理解技巧**：阻塞进程 `counter` 大 → 被优先调度 → 快速响应 I/O → 用户体验好。这是 Linux 0.11 照顾前台进程的核心机制

## ✅ 知识点4: `counter` 的多重作用与总结

**回顾一下 `counter`**
- `counter` 用**一个变量**同时实现了多种调度目标，代码简单高效

    | 作用 | 说明 |
    |:---|:---|
    | **保证响应时间的界** | 轮转调度，每个进程最多等待一个时间片周期 |
    | **照顾 I/O 进程（前台进程）** | I/O 阻塞后 `counter` 不被消耗，就绪后 `counter` 大 → 优先调度 |
    | **后台进程调度** | CPU 密集型进程 `counter` 消耗快，重新计算后按 `priority` 轮转，近似 **SJF（`Shortest Job First`）** |

- **初始值**：$c(0) = p$（等于静态优先级）
- **递推公式**：$c(t) = \frac{c(t-1)}{2} + p$
- **稳态分析**：$c(\infty) = p + \frac{p}{2} + \frac{p}{4} + ... = 2p$（等比级数收敛）


> 💡 **理解技巧**：一个进程如果长期 CPU  bound，`counter` 很快耗尽，重新计算后获得新的时间片；如果经常 I/O bound，`counter` 积累得越来越高，自然获得更高优先级
> 🔄 **知识关联**：这是操作系统设计中"简单即美"的典型——用一个变量解决多个问题，而非维护复杂的数据结构

## 🔑 核心要点总结
1. `schedule()` 选 `counter` 最大的 `TASK_RUNNING` 进程，若全为0则重新计算后再选
2. `counter` 是**时间片**：时钟中断递减，保证响应时间的界（轮转调度）
3. `counter` 是**动态优先级**：阻塞进程 `counter` 不消耗，就绪后更大 → 优先调度 I/O 进程（前台）
4. `counter` 重新计算 $c = c/2 + p$ 既保留了历史信息（旧 counter 的一半），又引入静态优先级 $p$
5. **一个变量折中多种需求**：响应时间、I/O 优先、后台轮转，代码极简但功能完备

## 📌 考试速记版
- **核心代码**：找最大 counter → `if(c) break` → 否则 `counter = counter/2 + priority` → `switch_to(next)`
- **双重作用**：时间片（RR 保证响应）+ 动态优先级（I/O 优先）
- **关键公式**：$c(t) = c(t-1)/2 + p$
- **易混淆概念对比**：
  - `priority`：静态，创建时设定，不变
  - `counter`：动态，运行时变化，既是时间片又是优先级
- **常见考试陷阱**：`counter >> 1` 是右移不是左移；阻塞进程优先级"变高"是因为 `counter` 没被消耗，不是被增加了

**记忆口诀**：counter 两用巧，时间片里藏优先级，I/O 堵完回来counter大，前台进程笑哈哈！
